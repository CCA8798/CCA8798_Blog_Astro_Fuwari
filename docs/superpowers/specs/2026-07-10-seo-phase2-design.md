# SEO Phase 2: WebSite/Person JSON-LD, hreflang, Sitemap, Performance SEO

## Overview

Phase 2 of SEO external search enhancements. Four independent work items, all purely frontmatter/meta-tag/config changes — no new dependencies, no runtime logic.

## Design Decisions (User-Approved)

| Decision | Choice | Notes |
|----------|--------|-------|
| JSON-LD identity | `Person` (personal blog) | |
| JSON-LD `logo` | → `image` | `Person` schema has no `logo` property |
| `sameAs` filtering | Only external links | Filter `profileConfig.links` to http/https only |
| SearchAction | **Omitted** | No dedicated search page; Pagefind is client-side only |
| hreflang | `x-default` only | Single language site |
| Sitemap | Medium customization | priority + changefreq + filter |
| Archive priority | 0.4 (not 0.6) | Archive pages are low importance |
| Preconnect | API origins only | open-meteo.com, hitokoto.cn |
| Font preload | **Skipped** | Astro processes fonts with hashed URLs — cannot hardcode |

## 1. WebSite + Person JSON-LD

### Component: `src/components/widget/SeoJsonLd.astro`

New Astro component that renders a `@graph` JSON-LD script with two entries:

**WebSite:**
```json
{
  "@type": "WebSite",
  "name": "CCA8798",
  "url": "https://www.cca8798.com",
  "description": "捌拐玖捌的家"
}
```
No SearchAction — Pagefind is a client-side search overlay, no URL-based search page.

**Person:**
```json
{
  "@type": "Person",
  "name": "不会起名の捌拐玖捌🍥",
  "url": "https://www.cca8798.com",
  "image": {
    "@type": "ImageObject",
    "url": "https://www.cca8798.com/assets/images/demo-avatar.png"
  },
  "sameAs": [
    "https://x.com/CCA8798suki",
    "https://space.bilibili.com/688231358",
    "https://github.com/CCA8798"
  ]
}
```

Key details:
- `image` not `logo` — `Person` schema has no `logo`. Using `ImageObject` wrapper per Schema.org best practice.
- `sameAs` filters `profileConfig.links` to only entries where `url` starts with `https://` — excludes internal links like `/about`.
- No SearchAction — site has no `/search` URL endpoint. Pagefind is a client-side modal.

### Injection point
`Layout.astro` → `<head>`, after the OG meta tags block.

### Data sources (existing, no type changes)
- `src/config.ts`: `siteConfig` (title, subtitle), `profileConfig` (name, avatar, links)

## 3. Sitemap Customization

### Change: `astro.config.mjs`

Modify the existing `sitemap()` integration config:

```js
sitemap({
  filter: (page) => {
    const { pathname } = new URL(page);
    const excluded = ['/admin/', '/login/', '/register/', '/404/'];
    return !excluded.some(p => pathname.startsWith(p));
  },
  serialize: (item) => {
    const url = new URL(item.url);
    if (url.pathname === '/' || url.pathname === '') {
      item.priority = 1.0;
      item.changefreq = 'daily';
    } else if (url.pathname.startsWith('/posts/')) {
      item.priority = 0.8;
      item.changefreq = 'weekly';
    } else if (url.pathname.startsWith('/archive/')) {
      item.priority = 0.4;
      item.changefreq = 'monthly';
    } else if (url.pathname.startsWith('/about/')) {
      item.priority = 0.5;
      item.changefreq = 'monthly';
    } else {
      item.priority = 0.4;
      item.changefreq = 'monthly';
    }
    return item;
  },
})
```

Key changes from original design:
- **No `SITE` variable** — uses `new URL(page).pathname` for safe matching.
- **Precise filter** — `pathname.startsWith()` instead of `.includes()` prevents false matches (e.g., `/about/admin-info/`).
- **Archive priority 0.4** — archive pages are low-value for SEO.
- **No `customPages`** — lotteryTimes, status-history, etc. all live in `src/pages/*.astro`, auto-generated by Astro.

## 4. Performance SEO

### Preconnect: `src/layouts/Layout.astro`

Add to `<head>`:
```html
<link rel="preconnect" href="https://open-meteo.com">
<link rel="preconnect" href="https://v1.hitokoto.cn">
```

### Font preload — SKIPPED

Rationale:
- `@fontsource/roboto` is imported as CSS → Astro bundles it with hashed filenames; URL unknown at template time.
- Custom Chinese fonts (`YangRenDongZhuShiTi-Semibold.woff2`, etc.) are configured via Astro's `fonts` config → output goes to `dist/client/_astro/*.woff2` with content hashes.
- Cannot hardcode preload URLs for hashed assets.
- Astro's font system + `font-display: swap` (via @fontsource) handles loading adequately.
- Existing commented-out font preloading code (Layout.astro lines 279-290) can be removed to avoid confusion.
