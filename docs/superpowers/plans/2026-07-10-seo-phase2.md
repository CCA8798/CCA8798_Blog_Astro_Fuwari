**Files:**
- Create: `src/components/widget/SeoJsonLd.astro`
- Modify: (none yet)

- [ ] **Step 1: Write the SeoJsonLd component**

Create `src/components/widget/SeoJsonLd.astro`:

```astro
---
import { profileConfig, siteConfig } from "../../config";

const siteUrl = Astro.site;
if (!siteUrl) throw new Error("Astro.site is not defined");

const webSite = {
  "@type": "WebSite",
  name: siteConfig.title,
  url: siteUrl.href,
  description: siteConfig.subtitle,
};

const sameAs = profileConfig.links
  .filter((link) => /^https?:\/\//.test(link.url))
  .map((link) => link.url);

const person = {
  "@type": "Person",
  name: profileConfig.name,
  url: siteUrl.href,
  image: {
    "@type": "ImageObject",
    url: profileConfig.avatar
      ? new URL(profileConfig.avatar, siteUrl).href
      : undefined,
  },
  sameAs: sameAs.length > 0 ? sameAs : undefined,
};

const graph = {
  "@context": "https://schema.org",
  "@graph": [webSite, person],
};
---

<script type="application/ld+json" set:html={JSON.stringify(graph, null, 2)} />
```

Key points:
- `Person.image` uses `ImageObject` wrapper (not `logo` — `Person` schema has no `logo`)
- `sameAs` filters to external URLs only (http/https protocol check)
- No `SearchAction` — site has no URL-based search page
- Uses `set:html` to prevent any potential HTML escaping issues
- Guards against undefined `Astro.site` at build time
- Filters out undefined properties (`sameAs` empty, `image.url` undefined) — `JSON.stringify` drops `undefined` values automatically

### Task 4: Customize sitemap configuration

**Files:**
- Modify: `astro.config.mjs`

- [ ] **Step 1: Replace default sitemap() with customized version**

In `astro.config.mjs` (around line 139), change:
```js
sitemap(),
```
to:
```js
sitemap({
  filter: (page) => {
    const { pathname } = new URL(page);
    const excluded = [
      '/admin/',
      '/login/',
      '/register/',
      '/404/',
      '/lotteryTimes/',
      '/status-history/',
    ];
    return !excluded.some(p => pathname.startsWith(p));
  },
  serialize: (item) => {
    const url = new URL(item.url);
    if (url.pathname === '/' || url.pathname === '') {
      item.priority = 1.0;
      item.changefreq = 'daily';
    } else if (url.pathname.startsWith('/posts/')) {
      item.priority = 0.8;
      item.changefreq = 'weekly';
    } else if (url.pathname.startsWith('/archive/')) {
      item.priority = 0.4;
      item.changefreq = 'monthly';
    } else if (url.pathname.startsWith('/about/')) {
      item.priority = 0.5;
      item.changefreq = 'monthly';
    } else {
      item.priority = 0.4;
      item.changefreq = 'monthly';
    }
    return item;
  },
}),
```

Key points:
- Excluded paths: `/admin/`, `/login/`, `/register/`, `/404/`, `/lotteryTimes/`, `/status-history/`
- Uses `pathname.startsWith()` for precise matching (not `.includes()`)
- Root path `/` gets priority 1.0
- Posts get 0.8, about 0.5, everything else 0.4
- No `customPages` needed — all pages are in `src/pages/` and auto-generated
- No external `SITE` variable needed — all matching done via `new URL(page).pathname`
